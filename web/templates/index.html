Inspector — las 7 partes del TP en vivo.

Alfabeto

Σ = { '.', '-', ' ', '/', '\n', '"' }

Gramática EBNF

programa     ::= { sentencia } ;
sentencia    ::= declaracion | asignacion | mostrar | si | mientras ;
declaracion  ::= "VAR" identificador "ASIG" expresion "\n" ;
asignacion   ::= identificador "ASIG" expresion "\n" ;
mostrar      ::= "MOSTRAR" expresion "\n" ;
si           ::= "SI" expresion "\n" { sentencia } [ "SINO" "\n" { sentencia } ] "FIN" "\n" ;
mientras     ::= "MIENTRAS" expresion "\n" { sentencia } "FIN" "\n" ;
expresion    ::= comparacion ;
comparacion  ::= aritmetica [ ("IGUAL"|"DIST"|"MEN"|"MAY") aritmetica ] ;
aritmetica   ::= termino { ("MAS"|"MENOS") termino } ;
termino      ::= factor { ("POR"|"DIV") factor } ;
factor       ::= numero | texto | "VERDADERO" | "FALSO" | identificador
              | "(" expresion ")" ;

Derivación — VAR X ASIG 10

sentencia ⇒ declaracion
         ⇒ "VAR" identificador "ASIG" expresion
         ⇒ "VAR" "X" "ASIG" numero
         ⇒ "VAR" "X" "ASIG" "10"

Trace en vivo — Morse → tokens

Tipo de parser

Recursivo descendente, LL(1). Cada no-terminal se decide con un único token de lookahead — las sentencias por su keyword inicial (VAR / MOSTRAR / SI / MIENTRAS) o por un IDENT en la asignación. Las expresiones usan precedencia por niveles para evitar recursión por izquierda. Implementación a mano, sin parser generators.

Intérprete + ElevenLabs

La fase de síntesis del TP es el intérprete tree-walking: recorre el AST, ejecuta cada sentencia y captura cada MOSTRAR en un buffer. La salida acumulada se manda opcionalmente a la API de ElevenLabs y se convierte en un output.mp3 narrado con voz humana. 

archivo.morse → Lexer → Parser → AST → Semántico → Intérprete → output.mp3

CLI:

python main.py examples/factorial.morse
python main.py examples/factorial.morse --tts

Bitácora de uso de IA

El proyecto se desarrolló con asistencia de Claude Code (Opus 4.7). Toda la bitácora está en docs/uso_ia.md: prompts importantes, componentes generados con IA, errores detectados en respuestas y correcciones manuales.

  • Diseño del lenguaje y elección de operadores como palabras Morse (no símbolos).
  • Estrategia del lexer en dos pasos (decodificación Morse + clasificación).
  • Parser recursivo descendente con un método por no-terminal.
  • Intérprete tree-walking + wrapper de ElevenLabs con cliente inyectable.
  • Decoder de audio puro con numpy/scipy (envelope RMS + Otsu + RLE).
